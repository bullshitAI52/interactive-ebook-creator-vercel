# 🎯 互动点读电子书制作器 (Interactive E-book Creator)

这是一个**所见即所得**的网页版点读机制作工具。哪怕不懂代码，也可以像做 PPT 一样，为您孩子制作专属的点读教材。

## ✨ 核心亮点

*   **⚡️ 极速预览**：上传图片和音频，**立即**就能在网页里点读测试，无需等待。
*   **🖱️ 简单易用**：上传图片 -> 拖拽按钮 -> 绑定声音，三步搞定。
*   **🤖 自动朗读**：(新功能) 支持自动播放当前页所有声音，并自动翻页，变身“磨耳朵”神器。
*   **📱 多端通用**：制作好的内容，在电脑、iPad、手机浏览器上都能完美运行。

---

## 🚀 快速开始

### 第一步：启动工具
由于浏览器的安全限制，直接双击 html 文件可能无法播放音频。建议在文件夹内运行一个简单的本地服务器：

```bash
# Mac/Linux 终端
cd /path/to/project
python3 -m http.server 8080
```

然后打开浏览器访问：
*   **编辑器 (制作)**: [http://localhost:8080/editor.html](http://localhost:8080/editor.html)
*   **播放器 (使用)**: [http://localhost:8080/index.html](http://localhost:8080/index.html)

## 🛠️ 如何制作电子书 (编辑器指南)

1.  **添加页面**：
    *   在左侧列表点击 `+` 增加新页面。
    *   点击 **"上传图片"**，选择一张绘本照片或教材扫描图（支持 JPG/PNG）。
    *   选择页面方向（纵向/横向）。

2.  **添加点读按钮**：
    *   点击工具栏的 **"添加按钮"** (或右下角 + 号)。
    *   **拖拽**画布上的圆圈按钮到想要的位置（比如苹果图案上）。
    *   **双击**按钮（或点击底部列表的 Edit），在弹窗中点击 **"浏览..."** 选择对应的 MP3 音频文件。
    *   *提示：此时您已经可以直接点击底部列表的 Play 试听了！*

3.  **整理顺序 (可选)**：
    *   在底部列表使用 **↑ / ↓** 调整按钮顺序（这将决定自动朗读时的播放顺序）。

4.  **保存成果**：
    *   按 **`Ctrl + S`** (或 Cmd + S) 保存配置。
    *   浏览器会自动下载一个 `book.json` 文件。
    *   **重要**：请用这个新下载的 `book.json` 替换掉项目根目录里原来的那个。

## 📂 文件归档 (至关重要！)

编辑器里的“即时预览”是为了方便您快速制作。当您制作完成后，为了让别人也能看到，或者下次打开还能用，请务必执行以下**归档操作**：

1.  打开项目里的 **`images`** 文件夹，把您刚才上传过的所有图片复制进去。
2.  打开项目里的 **`audio`** 文件夹，把您刚才用到的所有 MP3 音频复制进去。

> **原理**：`book.json` 里只记录了文件名（如 "cat.mp3"）。只有当文件真实存在于 `audio/cat.mp3` 时，播放器才能在任何时候找到它。

## 🎧 如何使用 (播放器指南)

打开 `index.html`，即刻开始学习：

*   **点读模式**：用鼠标或手指点击页面上的气泡，立刻播放对应发音，且按钮会高亮显示。
*   **自动朗读模式**：点击底部的 **"自动朗读"** 按钮。
    *   系统会自动按顺序播放当前页的所有内容。
    *   当前页读完后，会自动翻页继续朗读。
    *   适合睡前或磨耳朵场景。

### ⌨️ 快捷键

| 按键 | 功能 |
| :--- | :--- |
| **空格 (Space)** | 暂停 / 继续播放 |
| **← / →** | 上一页 / 下一页 |
| **F** | 进入/退出 全屏模式 |
| **T** | 切换 深色/浅色 主题 |
| **E** | 快速跳转到编辑器 |

## ❓ 常见问题

**Q: 为什么我保存后，重新打开图片不见了？**
A: 请确认您是否将图片文件复制到了 `images` 文件夹中。编辑器只保存“引用路径”，不保存文件本身。

**Q: 怎么调整按钮大小？**
A: 目前统一为标准指触大小，适配 iPad 操作。后续版本可增加调节功能。

**Q: 可以在手机上用吗？**
A: 可以！将整个文件夹上传到 GitHub Pages 或者是任何静态网站托管服务，用手机访问网址即可。
